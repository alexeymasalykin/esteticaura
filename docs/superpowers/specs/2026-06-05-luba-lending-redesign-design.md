## 1. Цель

Полный редизайн WebGL-лендинга бьюти-студии **ESTETICAURA Love** до уровня топ-студии в эстетике
**золото-нуар** (золото на чёрном). Редизайн затрагивает всё: дизайн-систему, верстку, контент
(перевод на русский) и 3D-сцену. Параллельно проект приводится в рабочее состояние — текущая
сборка сломана.

### Решения, зафиксированные на этапе brainstorming
- **Глубина:** полный редизайн, включая 3D-сцену.
- **Направление:** золото-нуар, усиленный до студийного уровня (не смена палитры).
- **Язык контента:** русский. Бренд `ESTETICAURA` остаётся латиницей.
- **3D-концепция:** развить существующую метафору «капля → кристалл» (вода/драгоценность = красота),
  улучшить материалы, морфинг и отражения.
- **Подход к реализации:** A — поэтапно (реанимация → дизайн-система → контент → 3D-апгрейд).

## 2. Исходное состояние и проблемы

Стек: Vite 5, чистый JS (ESM), Three.js 0.160, GSAP 3.13 (+ ScrollTrigger), lil-gui.
Архитектура «Experience» (ядро + World + UI-слой).

**Критичные проблемы (подтверждены `vite build`):**
1. **`src/webgl/World/Emblem.js` синтаксически сломан** — тело `setGeometry()` затёрто обрывками
   свойств материала; геометрия и текстура не создаются. **Сборка падает именно здесь.**
2. **Отсутствуют ассеты:** код грузит `/logo.jpg` (текстура эмблемы) и `/music.mp3` (аудиоплеер);
   в `public/` только `vite.svg`.
3. **Debug-артефакт:** красный wireframe-куб в сцене (`World.js:21–26`).
4. **Рассинхрон стилей:** `index.html` переопределяет `style.css` инлайн-атрибутами `style="..."`.
5. **Мусор стартера:** `src/counter.js`, `src/javascript.svg`, favicon `vite.svg`.
6. **Кастомный курсор** багуется на тач-устройствах (висит в углу, нет медиа-гарда).
7. **AudioPlayer** показывает `alert()` при отсутствии трека — плохой UX.

## 3. Дизайн-система (фундамент)

Расширение существующих токенов `style.css`. Всё на CSS-переменных. Ключевой приём студийного
вида — **контраст трёх уровней глубины**: фон → поверхность → золотой акцент.

### Цвет
```css
--bg:            #050505;  /* базовый фон сцены */
--surface:       #0E0E0F;  /* карточки, попапы — чуть светлее фона */
--surface-hi:    #16161A;  /* поднятые элементы, hover */
--gold:          #D4AF37;  /* основной акцент */
--gold-light:    #F7E7CE;  /* заголовки, блики (шампань) */
--gold-dark:     #8A6E2F;  /* обводки, тени, scrollbar */
--text:          #F5F3EE;  /* основной текст (мягче чистого белого) */
--text-muted:    rgba(245,243,238,.66);
--text-faint:    rgba(245,243,238,.55);  /* WCAG: ≥3:1 для подписей (было .40 — проваливало) */
--border:        rgba(212,175,55,.18);
--border-strong: rgba(212,175,55,.45);
```
Отличие от текущего: карточка — **сероватая поверхность `--surface` на чёрном фоне**, а не белая
полупрозрачная плёнка. Контраст «фон → surface → золотая кнопка» отделяет студийный вид от
«нейросетевого».

**Правило контраста (WCAG, обязательно — тёмная тема легко проваливает):**
- `--gold #D4AF37` как **цвет текста** — только для крупного (заголовки `--fs-h2`+ и мета-цифры),
  где достаточно 3:1. Для body и подписей золото текстом **не использовать** — брать `--text` /
  `--text-muted`.
- Золотая кнопка (заливка `--gold` + текст `--bg`) — контраст высокий, ок.
- Каждую пару foreground/background проверять: body ≥ 4.5:1, вторичный/крупный ≥ 3:1.

### Типографика (4 уровня иерархии, адаптив через `clamp`)
| Токен | Размер | Шрифт | Назначение |
|---|---|---|---|
| `--fs-display` | `clamp(3rem, 8vw, 7rem)` | Playfair Display | hero-заголовок |
| `--fs-h1` | `clamp(2.2rem, 5vw, 3.5rem)` | Playfair Display | секции |
| `--fs-h2` | `clamp(1.5rem, 3vw, 2.2rem)` | Playfair Display | подзаголовки, мета-цифры |
| `--fs-h3` | `clamp(1.1rem, 2vw, 1.4rem)` | Inter 600 | карточки |
| `--fs-body` | `1rem`–`1.05rem` | Inter 400 | текст |
| `--fs-caption` | `0.8rem`, uppercase, `letter-spacing:.15em` | Inter | мета, навигация |

Цвета: заголовки `--gold-light`, тело `--text`, мета `--text-faint`.
Line-height: заголовки `1.1`, тело `1.6`.

### Spacing (8-px шкала)
`--space-1:4px · -2:8 · -3:16 · -4:24 · -5:40 · -6:64 · -7:96 · -8:128`. Только из шкалы.

### Радиусы / тени / движение
```css
--radius-sm:8px · --radius:16px · --radius-lg:24px · --radius-pill:999px
--shadow-card: 0 20px 40px rgba(0,0,0,.5);
--glow-gold:   0 0 24px rgba(212,175,55,.35);
--dur-fast: 150ms · --dur: 250ms · --ease-out: cubic-bezier(.16,1,.3,1)
```
Все hover/state-переходы (кнопки, карточки) — в диапазоне **150–300ms**, `ease-out` на вход.

### Числа (tabular)
Цены «от N ₽», мета-цифры (8 лет, 5000+) и любые числовые данные — с
`font-variant-numeric: tabular-nums`, чтобы не было дёргания ширины.

### Touch-targets
Все интерактивные элементы (кнопки, ссылки навигации, иконки соцсетей, бургер, поля/`select`
формы) — минимальная зона **≥ 44×44px**; мелкие иконки добирают зону через padding/`hitSlop`.

### Кнопки — две иерархии
- **Primary** (`.btn`): заливка `--gold`, текст `--bg`; hover — `--glow-gold` + лёгкий подъём.
  Главное действие (Записаться).
- **Secondary** (`.btn--ghost`): прозрачная, обводка `--border-strong`, текст `--gold`;
  hover — заливка золотом. Второстепенное (Услуги, Подробнее).

### Слои (z-index)
`canvas:-1 · content:10 · nav:100 · cursor:9998 · preloader:10000`.

## 4. Контент и структура страницы

Контент на русском, осмысленный (не lorem). Бренд `ESTETICAURA` латиницей. Каждая секция несёт
смысл — пустых экранов нет. Финальные тексты пишутся на этапе реализации; здесь — каркас и смысл.

1. **Навигация** (fixed, `mix-blend-mode:difference`): `ESTETICAURA` · меню «О студии · Услуги ·
   Запись · Контакты» · кнопка **«Записаться»** (primary, всегда видна). На mobile — бургер.
2. **Hero:** заголовок `Esteticaura Love` (золотой shine), подзаголовок *«Где красота становится
   искусством»*, кнопки **Записаться** (primary) + **Услуги** (ghost). Фон — 3D-капля.
3. **О студии:** заголовок *Наша философия*; 2–3 предложения о холистическом подходе; **мета-строка
   из 3 цифр** (`8 лет · 5000+ клиентов · 12 мастеров`) в `--fs-h2` золотом + подпись `--fs-caption`.
4. **Услуги:** заголовок *Услуги*; карточки на `--surface` — **Косметология · Массаж · Эстетика ·
   Уход** (адаптивная сетка 4→2→1). Каждая: номер/иконка, `--fs-h3` название, описание `--text-muted`,
   строка *«от N ₽»* — цвет `--text` + `tabular-nums` (не золотом: мелкий текст золотом проваливает
   WCAG), ссылка-ghost «Подробнее».
5. **Социальное доказательство** (новое — сильнейший конверсионный блок для бьюти): заголовок
   *Нам доверяют*. Два под-блока:
   - **Работы до/после** — сетка карточек с парами изображений (заглушки с `aspect-ratio`, без CLS);
     подпись услуги. Слайдер/сравнение опционально.
   - **Отзывы** — 3–5 карточек: фото + имя + услуга + текст. По паттерну «social proof перед CTA».
   - В конце блока — **дублирующий primary-CTA «Записаться»** (паттерн «CTA после доверия»).
6. **Запись** (главная конверсия): заголовок *Записаться на приём*; форма на `--surface` —
   **Имя · Телефон · Услуга (select) · «Отправить заявку»**.
   - Поля: `<label>` видимый; имя — `autocomplete="name"`; телефон — `type="tel"
     inputmode="tel" autocomplete="tel"` (правильная мобильная клавиатура); высота полей ≥ 44px.
   - Валидация **на blur** (не на каждый кейстрок), ошибка под полем, `role="alert"`/`aria-live`.
   - Состояния: отправка (спиннер на кнопке) → успех/ошибка (не `alert`).
   - Отправка — **ссылка на WhatsApp/Telegram** или `mailto` (без бэкенда).
7. **Контакты:** адрес, телефон, часы работы, иконки соцсетей (Instagram/Telegram/WhatsApp,
   `aria-label`, зона ≥ 44px), email; опц. статичная мини-карта-заглушка (изображение, без
   тяжёлого встраивания).
8. **Футер:** лого, копирайт, политика, дубль соцсетей.

Мета-цифры в секции «О студии» — крупные (`--fs-h2`), поэтому золото допустимо (3:1) + `tabular-nums`.
Все inline-стили из `index.html` переносятся в классы дизайн-системы.

## 5. 3D-сцена и хореография

Развитие «капля → кристалл», доведение до студийного уровня. Главные рычаги «вау» —
**environment-отражения + bloom**.

### Хореография по скроллу
| Секция | 3D-состояние |
|---|---|
| Hero | **Капля** — дорогое стекло, медленное дыхание, золотые блики окружения |
| О студии | Капля парит, золотая пыль сгущается |
| Услуги | **Морфинг капля → кристалл**: капля сжимается-вращается-распыляется → кристалл собирается |
| Социальное доказательство | Кристалл медленно вращается на грани, отражения играют |
| Контакты/футер | Кристалл оседает, пыль успокаивается |

### Технические апгрейды
1. **Environment map без внешних ассетов:** `THREE.RoomEnvironment` + `PMREMGenerator` →
   настоящие PBR-отражения на капле и кристалле без скачивания HDRI.
2. **Bloom-постпроцессинг:** `EffectComposer` + `UnrealBloomPass` → золотое свечение частиц и бликов.
   Только на десктопе (дорогой проход).
3. **Материалы:**
   - Капля: `MeshPhysicalMaterial`, `transmission:1`, `ior:1.45`, `clearcoat:1`, лёгкий `iridescence`.
   - Кристалл: золотой металл `metalness:1, roughness:.15`, `flatShading`, env-отражения.
4. **Морфинг — crossfade через GSAP** (надёжнее morph-targets): капля распыляется → кристалл
   собирается, с «вспышкой» частиц на переходе.
5. **Частицы → золотая пыль:** круглая alpha-маска вместо квадратных точек, мягкое мерцание;
   5000 на десктопе.
6. **Тон-маппинг:** `ACESFilmic` (вместо Reinhard), exposure ~1.1.

### Чистка 3D
- **`Emblem.js` удаляется** (сломан); hero-объект = капля; лого `ESTETICAURA` — HTML-текст в
  навигации/прелоадере. Убирает зависимость от `logo.jpg`.
- **Debug-куб удаляется.**
- `MorphShapes.js` разбивается на `Droplet.js` + `Crystal.js`.
- **`Resources.js` удаляется.** Внешних ассетов для async-загрузки больше нет (env map —
  процедурный `RoomEnvironment`, материалы/частицы процедурные). Готовность сцены = синхронная
  инициализация `World`; прелоадер скрывается по событию готовности `World` + таймаут-страховка.
- **`Lights.js` вливается в `Environment.js`** и удаляется как отдельный файл (свет + env в одном месте).

## 6. Перформанс, адаптив, доступность

### Адаптив
- Брейкпоинты: mobile `<640` · tablet `640–1024` · desktop `>1024`.
- Навигация: бургер-overlay на mobile; кнопка «Записаться» всегда видна.
- Сетка услуг 4→2→1; форма записи в одну колонку на mobile.

### Доступность
- **`prefers-reduced-motion`** (High-severity: ScrollTrigger/параллакс вызывают тошноту) →
  - Three.js не грузится, статичный постер;
  - **scroll-хореография ScrollTrigger отключается** — секции отображаются обычным потоком, без
    скролл-скраба и морфинга;
  - параллакс камеры/курсора и `shine`-анимация заголовка выключены;
  - hover-переходы остаются (они не вестибулярные).
- **Кастомный курсор — только desktop** (`@media (hover:hover) and (pointer:fine)`). Фикс тач-бага.
- Фокус-стейты (золотая обводка) на всех интерактивных; форма доступна с клавиатуры;
  `aria-label` на иконках/бургере; `label` для полей.
- Контраст по WCAG AA.

### Перформанс (бюджеты `~/.claude/rules/web-performance.md`)
- **Осознанное отклонение:** Three.js (~150kb gzipped) + GSAP не укладываются в бюджет лендинга
  < 150kb JS. Принято ради 3D. Митигация ниже.
- **Динамический импорт** Three.js + постпроцессинга после первого рендера (hero-текст виден сразу).
- **Fallback-постер** (CSS радиальный градиент + зерно) для mobile / `prefers-reduced-motion` /
  слабый GPU — Three.js там не качается вообще.
- Bloom только на десктопе; средние устройства — сцена без bloom.
- `IntersectionObserver` — пауза рендера вне вьюпорта.
- Шрифты: `preconnect` + `font-display:swap` + preload критичного веса Playfair.
- Цель: **FCP < 1.5s** при последующей догрузке сцены.

### Загрузочные / пустые состояния
- Прелоадер: стили в систему, событие `ready` + таймаут-страховка (без «магического» 5s).
- Форма: спиннер на кнопке при отправке, экраны успех/ошибка (не `alert`).
- **AudioPlayer удаляется** (зафиксировано): на бьюти-лендинге звук неуместен, лишний вес и риск.

## 10. Критерии приёмки

1. `npm run build` проходит без ошибок; `npm run dev` запускает рабочую сцену.
2. Никаких inline-стилей в `index.html`; все стили — через дизайн-систему.
3. Контент на русском; присутствуют все **8 секций**, включая **социальное доказательство**
   (отзывы / до-после) и работающую форму записи с валидацией на blur.
4. 3D-сцена: капля с env-отражениями в hero, морфинг в золотой кристалл по скроллу, bloom на десктопе.
5. На mobile / `prefers-reduced-motion`: Three.js не загружается, показан статичный постер;
   **ScrollTrigger-хореография и параллакс отключены**; бургер-меню работает; кастомный курсор отключён.
6. Удалены: `Emblem.js`, `MorphShapes.js`, `Resources.js`, `Lights.js`, `AudioPlayer.js`, debug-куб,
   `counter.js`, `javascript.svg`.
7. FCP < 1.5s (hero виден до догрузки сцены).
8. Доступность: фокус-стейты, `aria-label` на иконках/бургере, `label` у полей,
   **контраст WCAG AA проверен** (золото-текст только для крупного; `--text-faint` ≥ 3:1).
9. Полировка: цены/цифры с `tabular-nums`; все touch-targets ≥ 44px; поле телефона `type="tel"`;
   дублирующий primary-CTA после блока доверия.
