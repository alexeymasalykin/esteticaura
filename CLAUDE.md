# luba_lending — ESTETICAURA Love

Проектные инструкции для агента. Дополняют глобальный `~/.claude/CLAUDE.md`.

## Что это
Премиальный **WebGL-лендинг бьюти-студии ESTETICAURA Love** в эстетике **золото-нуар**
(золото `#D4AF37` на чёрном `#050505`). Идёт **полный редизайн** по утверждённой спеке.
Бренд `ESTETICAURA` — латиницей, контент — на **русском**.

## Стек
Vite 5 · vanilla JS (ESM, без фреймворка) · Three.js r160 · GSAP 3.13 (+ ScrollTrigger) · lil-gui.
Архитектура «Experience»: `Experience` (ядро) → `World` → 3D-объекты + `Environment`; UI-слой в `src/ui/`.

## Жёсткие правила (не нарушать)
1. **Контраст WCAG.** Золото `#D4AF37` текстом — ТОЛЬКО для крупного (заголовки/цифры, ≥3:1).
   Body и подписи — `--text`/`--text-muted`, никогда золотом. Body ≥ 4.5:1.
2. **reduced-motion.** При `prefers-reduced-motion` — НЕ грузить Three.js (статичный постер),
   отключить ScrollTrigger-хореографию и параллакс. Hover-переходы оставить.
3. **Перформанс.** Three.js не влезает в бюджет лендинга (<150kb JS) — осознанно. Митигация:
   динамический импорт 3D после первого рендера; на mobile/слабом GPU — постер вместо сцены.
4. **Touch ≥ 44px**, форма: `type="tel"`, `autocomplete`, валидация на blur, `aria-live`.
5. **Числа** (цены, мета-цифры) — `font-variant-numeric: tabular-nums`.
6. **Никаких inline-стилей** в `index.html` — только классы дизайн-системы.

## 3D-параметры (из борда — стартовые)
Капля: `MeshPhysicalMaterial { transmission:1, thickness:0.5, roughness:0.05, ior:1.33, iridescence:0.2 }`
(roughness 0.2–0.6 пикселит — избегать). Кристалл: `metalness:1, roughness:0.15, flatShading`.
Env: `RoomEnvironment` + `PMREMGenerator` (без HDRI-файлов). Bloom: `RenderPass → UnrealBloomPass(0.4,1.2,0.1) → OutputPass`
(**OutputPass обязателен последним** — иначе ломает тон-маппинг). Селективный bloom НЕ нужен (UI вне WebGL).

- **Фаза A — Реанимация + чистка** ✅ ЗАВЕРШЕНА (`9b0fa63`): сборка зелёная, чистая минимальная сцена
  (капля + частицы + RoomEnvironment-свет), мусор и сломанный код удалены.
- **Фаза B — Дизайн-система + русский контент 8 секций + форма (TDD)** ✅ ЗАВЕРШЕНА (`b94710d`):
  дизайн-система золото-нуар, 8 RU-секций без inline-стилей, форма с blur-валидацией → Telegram,
  бургер, слайдер до/после, Vitest (13 тестов), gate `ui-ux-pro-max` пройден (2 a11y-фикса).
- **Фаза C — 3D-апгрейд** ✅ ЗАВЕРШЕНА (`7c392f5`): UnrealBloom (десктоп), ACESFilmic, lil-gui по
  `#debug`. Тюнинг заказчика: exposure 0.39, bloom 0.38/0.43.
  **⚠️ Пост-C редизайн 3D по фидбэку заказчика (`aaa8f6d`, `9aedcc6`):** капля и солид-кристалл
  УДАЛЕНЫ (transmission не рендерится под bloom / на слабом GPU). Текущая сцена — **«бриллиант из
  пыли»**: hero = поле золотой пыли, на ~1/3 скролла пыль слетается во вспышке в **форму бриллианта
  огранки** из 6300 светящихся частиц (`World` = `Environment` + `Particles`; `Particles` хранит
  scatter+diamond позиции, лерпит по скроллу). Без transmission → надёжно на любом GPU. Проверка —
  Playwright headless (см. ниже).
- **Фаза D — Перформанс/адаптив/a11y** ✅ ЗАВЕРШЕНА (`1f16fc1`): динамический импорт Three.js после
  пейнта (main bundle 600kB→80kB / 31kB gzip, Three.js в ленивом chunk), статичный золотой постер
  при `prefers-reduced-motion` / без WebGL (Three.js не качается), пауза рендера при скрытой вкладке,
  `decideMode` (TDD, +4 теста). Событийная ветка прелоадера ожила (carry-forward закрыт).
  **Пост-D правка адаптива (`7c73f4e`) по запросу заказчика:** mobile БОЛЬШЕ не уходит в постер —
  телефоны тоже рендерят 3D-бриллиант; `Particles` адаптивно ужимает форму бриллианта на узком/
  портретном аспекте (`fit = clamp(aspect/1.15)`) и режет число частиц на телефонах (3800 vs 6300).
  Постер остаётся фолбэком только для reduced-motion / без WebGL.
- **Графический апгрейд (2026-06-11)** ✅ по аудиту графики (план
  `docs/superpowers/plans/2026-06-11-luba-graphics-upgrade.md`): хореография в **три акта** —
  пыль → сборка бриллианта со вспышкой (opacity + `bloomPass.strength` 0.38→0.95→0.38) → бриллиант
  уменьшается, дрейфует вправо-вверх и гаснет до 0.32 (контент booking/contacts на спокойном фоне).
  Бриллиант `S=1.8`, центрован по Y, целиком влезает в кадр (dolly теперь НАРУЖУ `z 6→6.8`).
  Частицы на `ShaderMaterial`: морф в вершиннике (staggered по `aDelay`, CPU-цикл удалён),
  per-particle twinkle (`aPhase`), палитра gold/champagne/white, ~28% частиц сэмплируются по
  **рёбрам фасетов** (ярче/крупнее — огранка читается), дрейф пыли в hero, затухающий с прогрессом.
  Генерация — чистый модуль `diamondPoints.js` (TDD, +9 тестов). Композер ограничен DPR 1.5.
  В шейдере обязательны `#include <tonemapping_fragment>` / `<colorspace_fragment>` — паритет
  ACES на мобиле (без композера). Дополнительно: `uBoost` уравнивает плотность блёсток десктопа
  с мобильным видом; **кометный шлейф за курсором** (пыль стягивается к лучу курсора во
  view-space, голова/ленивый хвост; только `pointer: fine`, только в состоянии пыли; сила —
  в `#debug` → Dust/comet). Отвергнуто заказчиком: «осколки стекла» (жёсткие ромбы) — ретро-вид.

Планы B/C/D пишутся ПОСЛЕ завершения и проверки предыдущей фазы (детали опираются на факт).
На этапе кода — обязательный gate `ui-ux-pro-max` (контраст, a11y, touch).
